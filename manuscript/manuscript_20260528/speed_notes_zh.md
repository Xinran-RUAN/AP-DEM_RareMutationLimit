# 运行速度说明与建议参数

## 为什么现在比旧代码慢

旧 1D WKB 主程序中的典型参数为：

- `N_x = 5`，也就是 `x` 方向只有 `6` 个节点；
- `N_theta = 50`；
- 每一步需要求 `50` 个 `6 x 6` 的空间主特征值问题；
- 振幅 `W` 的未知量约为 `6 * 50 = 300`。

重构后为了更接近论文数值实验，一些脚本曾默认使用：

- `Nx = 200`，也就是 `x` 方向 `201` 个节点；
- `Ntheta = 128` 或 `256`；
- 每一步需要求 `128/256` 个 `201 x 201` 的空间主特征值问题；
- 振幅 `W` 的未知量约为 `201 * 128 = 25728`，Test 3 中为 `201 * 256 = 51456`。

因此慢的主要原因确实是离散点数比旧代码大很多，尤其是 `Nx` 大很多。WKB 求解器每一步都要计算所有 trait 点上的有效 Hamiltonian `H(theta)`，这个特征值计算是主要耗时来源。

## 建议用法

日常调试和看现象：

```matlab
run_main_1d
```

其中默认 `runMode = 'quick'`，参数为

```matlab
Nx = 50;
Ntheta = 64;
T = 10;
dt = 2e-3;
```

如果只是排错，可以把 `runMode` 改成 `'oldlike'`，接近旧代码规模。

正式论文数据：

```matlab
runMode = 'paper';
```

或者在各个 `run_test*.m` 里把

```matlab
quickMode = true;
```

改成

```matlab
quickMode = false;
```

## 代码中的速度优化

`+src/solve_H_fd_1d.m` 做了两个不改变离散模型的优化：

1. 预先构造梯形权重下的对称相似变换后的 `Lx`，避免在 `theta` 循环里重复做矩阵相似变换；
2. 对 `eigs` 使用上一个 `theta` 的主特征向量作为初始向量 `v0`，因为相邻 trait 的特征问题变化较小；小矩阵时直接使用 dense `eig`，避免 `eigs` 的迭代启动开销。

## 粗略复杂度判断

把 `Nx` 从 `50` 提到 `200`，空间特征值问题维度从 `51` 变成 `201`；把 `Ntheta` 从 `64` 提到 `256`，每步特征值问题数量又增加 4 倍。若同时把 `dt` 从 `1e-3` 改成 `5e-4`，步数再增加 2 倍。因此 Test 3 的正式参数天然会很慢。
